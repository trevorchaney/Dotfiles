--># Git Hooks maintained in repo

Location
- Hooks are kept in `git/.config/git/hooks` inside this repository.

Install
- From Git Bash (or any POSIX shell):

```sh
# run from repo root or give a path to the target repo
git/install-hooks.sh /path/to/target/repo
```

- From PowerShell:

```powershell
# run from repo root or give a path to the target repo
.\git\install-hooks.ps1 -Target "C:\path\to\repo"
```

Behavior
- The `pre-commit` hook scans staged hunks for two markers:
  - `@do-not-commit` — automatically aborts the commit if present in an added line.
  - `@review` — shows the hunk and prompts the committer to accept (`y`) or reject (`n`). Rejection aborts the commit.

Notes
- The installer sets `core.hooksPath` in the target repository to the hooks directory in this repo. This approach avoids copying files and keeps hooks centrally maintained.
- If you prefer copying hooks into `.git/hooks/`, copy the files manually or modify the installer script.
